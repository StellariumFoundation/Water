## üõ† Getting Started

### Prerequisites
*   [Go 1.24+](https://golang.org/dl/)

*   GCC / Clang (CGO is required for the Fyne GUI)
*   Linux: `libgl1-mesa-dev`, `libxcursor-dev`, `libxrandr-dev`, `libxinerama-dev`, `libxi-dev`, `libxxf86vm-dev`
*   macOS: Xcode Command Line Tools
*   Windows: MinGW-w64

### Installation
1.  **Clone the Repository**
    ```bash
    git clone https://github.com/StellariumFoundation/Water.git
    cd Water
    ```

2.  **Build & Run**
    ```bash
    make build
    ./bin/water          # Launches the gateway + Fyne GUI
    ./bin/water server   # Headless gateway only
    ./bin/water --version
    ```
    *A single binary starts both the backend gateway (port 7777) and the Fyne desktop GUI.*

3.  **Release builds for all platforms**
    ```bash
    make release         # Outputs to dist/
    ```



### Quick Start (Development)

Build and run the Go backend without the frontend (no Node.js required):

```bash
git clone https://github.com/StellariumFoundation/Water.git
cd Water

# Create mock assets and build Go binary
make build-dev

# Run the server directly
./bin/water-ai server
```

The server starts on `http://localhost:7777`.

### Full Build (with Frontend)

Build the complete application with the embedded Next.js frontend:

```bash
# Install frontend dependencies and build everything
make build

# Run the binary (auto-spawns background daemon + opens browser)
./bin/water-ai
```

### Running Tests

```bash
# Run all unit tests
make test

# Run tests with race detection
make test-race

# Generate HTML coverage report
make test-coverage
```

### Cross-Platform Release

```bash
# Build optimized binaries for all platforms (Linux, macOS, Windows)
make release
```

Release binaries are output to the `dist/` directory.

### Fyne GUI Client

A native desktop GUI client built with [Fyne](https://fyne.io/) is available on the [`gui`](https://github.com/StellariumFoundation/Water/tree/gui) branch:

```bash
git checkout gui
make gui
./bin/water-gui
```

> **Note:** The Fyne GUI requires CGO and platform-specific graphics libraries. See the `gui` branch README for details.
